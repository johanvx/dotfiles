# Reference: High-Signal Technical Writing (Tufte/Feynman-Compatible)

Use this guide during drafting and revision.

## Core principles

1. **Clarity over performance**: optimize for understanding, not impressive wording.
2. **Evidence over adjectives**: use measurements, mechanisms, constraints.
3. **High signal-to-noise**: each sentence must add information.
4. **First-principles explanations**: define assumptions and causality.
5. **Honest scope**: state limitations and uncertainty explicitly.
6. **Actionable conclusions**: end with decisions or next actions.

## Compression method for “too much context”

1. Define one takeaway sentence.
2. Filter notes by relevance to user constraints.
3. Convert selected items into 3-line cards:
   - Claim
   - Evidence
   - Implication
4. Draft from structure (not free-form).
5. Remove repetition and vague language.

## Sentence-level rules

- One sentence = one idea.
- Prefer: “X causes Y under Z conditions.”
- Avoid hype, vague intensifiers, and redundant framing.
- Keep connectors explicit when needed: “However,” “Therefore,” “Given this constraint,”.

## Overclaim prevention

Replace absolute claims unless fully justified:
- “proves” -> “indicates/suggests under tested conditions”
- “best” -> “competitive in evaluated settings”
- “general” -> “observed in tested configurations”

## Pre-publish checklist

- Can the takeaway be stated in one sentence?
- Are claims tied to evidence?
- Are assumptions and limits explicit?
- Are references present or clearly tagged as needed?
- Is there any sentence that only sounds smart but adds no content?
